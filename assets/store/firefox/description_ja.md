## 詳細説明（Description）

**あなたのパスワードは、あなた自身のストレージにしか保存されません。**

kura は、開発者が運用するサーバーを一切持たないパスワードマネージャーです。暗号化された vault ファイルを、あなた自身の S3 互換ストレージ（AWS S3 / Cloudflare R2 / MinIO / self-hosted 等）に単一ファイルとして保存します。

## 主な特徴

- **ゼロ知識暗号化** — 暗号化・復号はすべてお使いのデバイスで完結。開発者もストレージプロバイダも、あなたのデータを平文で見ることはできません。
- **サーバ不要** — 開発者側のサーバーやアカウント登録は存在しません。ベンダーロックインなし。
- **マルチプラットフォーム同期** — 同じ vault を、デスクトップ（Windows / macOS / Linux）、Android、Chrome / Firefox 拡張機能で共有できます。
- **オフラインファースト** — オフライン時もローカルキャッシュで動作し、接続回復時に自動同期。
- **自動コンフリクト解消** — 複数デバイスで編集した場合、タイムスタンプベースで自動マージ。データロストしません。
- **ログインフォーム自動入力** — 訪問中のサイトに対応するエントリを自動入力。URL 判定のみで、閲覧履歴は一切取得しません。
- **1Password からの移行** — 1pux エクスポート形式で 1Password のデータを取り込めます。

## セキュリティ

- **AES-256-GCM** による認証付き暗号化
- **Argon2id** によるメモリハード鍵導出
- **二層の鍵構造（KEK / DEK）** — マスターパスワード由来の KEK で DEK を暗号化
- **リカバリーキー** — マスターパスワードを忘れた場合の復旧手段として、セットアップ時に生成
- **オートロック** — 一定時間の無操作またはバックグラウンド移行で自動的にロック

## プライバシー

- テレメトリ、解析、広告、クラッシュレポート送信は一切ありません
- 通信先はあなたが指定した S3 エンドポイントのみ
- オープンソース（Apache License 2.0）

プライバシーポリシー: https://muranoya.github.io/kura.github.io/privacy-ja.html

## こんな方におすすめ

- 自分のパスワードは自分で管理したい
- クラウドパスワードマネージャーのサーバーに自分のデータを預けることに不安がある
- これから初めて自分専用のストレージを使ってみたい
- 既に AWS / Cloudflare R2 / MinIO 等を使っており、パスワード管理にも活用したい
- オープンソースで透明性のある実装を使いたい

## セットアップについて

S3 互換ストレージの準備は、公式ドキュメントの手順に沿って進められます。Cloudflare R2 など、個人利用なら無料枠または少額の料金で使えるサービスもあります。パスワードマネージャーを初めて使う方にも、既存サービスから自分のストレージへの移行を検討中の方にも適しています。

## 対応ストレージ

- AWS S3
- Cloudflare R2
- MinIO（self-hosted 含む）
- その他 S3 互換で Conditional Write（If-Match）に対応するサービス

## 注意事項

- kura は開発初期段階にあり、データ形式や機能は予告なく変更される可能性があります
- マスターパスワードとリカバリーキーの両方を紛失した場合、保存されたデータを復旧する方法はありません
- ストレージサービスの利用料金はユーザーの負担となります

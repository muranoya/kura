Search Combobox Skip

Tests that search combobox fields with Japanese aria-labels are correctly skipped from autofill (regression test for role="combobox" + aria-label="ユーザーを検索" bug).

Expected Behavior
  • Section 1 (Japanese search combobox): aria-label="ユーザーを検索", role="combobox" → no autofill dropdown
  • Section 2 (ASCII search combobox): aria-label="Search users" → no autofill dropdown (skipped via /search/i pattern)
  • Section 3 (normal login form): aria-label="ユーザー名" → dropdown appears on focus (correctly classified as username)
  • Focusing search inputs should do nothing
  • Focusing login inputs should show autofill dropdown with candidates
